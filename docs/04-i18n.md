# i18n

Locale order: **pl (primary) → en → ja → ko → uk**. The theme is string-complete from
day one; store content translations are enabled per-language later (Shopify Markets /
Translate & Adapt emit hreflang automatically).

## Discipline

- `locales/en.default.json` = storefront source of truth; `en.default.schema.json` = editor strings.
- Zero hardcoded strings in any `konbini-*` file (theme-check enforces in CI).
- All KONBINI keys under a top-level `"konbini"` object per file (additive → merge-safe).
- Every PR adding a key must include the `pl.json` translation.
- Upstream ships `pl`, `ja`, `ko` (+ schema variants). `locales/uk.json` is ours
  (storefront-only; `uk` is not an editor schema locale — fine, the client works in Polish).
- On upstream sync: diff `pl.json` against `en.default.json` for untranslated new keys.

## CJK typography contingencies (groundwork in Phase 1)

- `:lang(ja)` / `:lang(ko)` font stacks in `assets/konbini-overrides.css` — system CJK
  stacks or self-hosted subset Noto Sans JP/KR (watch CWV weight budget).
- `:lang(ko) { word-break: keep-all; }`; strict line-breaking for ja.
- `text-transform: uppercase` and airy letter-spacing are **Latin-scoped** via `:lang()` —
  uppercase is a no-op/harmful and wide tracking breaks legibility in CJK.
- No faux italics for CJK.
- `konbini.name_original` renders JP/KR glyphs on Polish pages **immediately** — inline
  spans must carry `lang="ja"`/`lang="ko"` and the fallback stack from day one.
